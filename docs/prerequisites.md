# 学前准备：概念速查（对齐学习计划）

面向「能开始写 LangChain/LangGraph 代码」的最小认知集，细节以 [LangChain 文档](https://docs.langchain.com/) 为准。

## LLM 与对话 API

- **Prompt（提示）**：你发给模型的指令与上下文；**多轮对话**即每次请求带上历史消息（system / user / assistant 等角色）。
- **Token**：模型按片段计费的单位；上下文有长度上限，长对话需要裁剪或摘要。
- **温度（temperature）**：越高输出越随机，越低越稳定；结构化任务通常偏低。
- **流式（stream）**：边生成边返回 token，适合 CLI/Web 实时展示。

## 工具调用（Tool / Function Calling）

- 模型在结构化格式里返回「要调用的工具名 + 参数」；**你的程序**负责真正执行 HTTP、查库、计算等，再把**工具结果**塞回对话，让模型继续推理。
- 与「在 prompt 里假装 JSON」不同：官方 tool 接口有约定 schema，便于校验与安全边界。

## RAG（检索增强生成）

- **流程**：从知识库检索相关片段 → **拼进 prompt 上下文** → 模型基于这些片段回答。
- **解决的问题**：模型不知道你私域文档、或需要引用最新资料时，用检索补全上下文，而不是只靠预训练记忆。

## Agent（典型 ReAct 形态）

- **循环**：模型**思考**下一步 → **选择是否调用工具** → **执行工具**得到观测 → 再思考，直到给出最终答案或达到终止条件。
- 与「单次问答」相比，Agent 适合需要多步查询、试错、组合多个能力的任务；同时要控制权限、超时与成本。
